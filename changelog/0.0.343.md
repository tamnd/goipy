# v0.0.343 — 2026-04-29

Generator / coroutine / async-generator introspection (F2)

Closes group **F2** from the v0.1.0 audit
(`notes/Spec/1500/1530_goipy_v01_gap_audit.md`). Spec at
`notes/Spec/1500/1537_goipy_v0043_generator_introspection.md`.

`*object.Generator` is the runtime type for plain generators,
coroutines, and async generators (separated by `Code.Flags`). None
of CPython's introspection attrs were exposed; `pytest-asyncio`,
`trio`, and `inspect.iscoroutine*` rely on `cr_running` /
`cr_frame` to gate test-fixture cleanup and silently no-op'd today.

- `object/generator.go`: new `Running bool` flag — surfaces through
  `gi_running` / `cr_running` / `ag_running`.
- `vm/generator.go`: `resumeGenerator` and `throwGenerator` set
  `Running = true` for the duration of the dispatch step.
- `vm/generator.go` `genIntrospect`: dispatches the right prefix
  (`gi_` / `cr_` / `ag_`) per `Code.Flags`, returns:
    - `*_frame` → `*object.TracebackFrame{Code}` while live, None
      after completion.
    - `*_code` → underlying `*object.Code`.
    - `*_running` → `Running` flag.
    - `gi_yieldfrom` / `cr_await` / `ag_await` → None (we don't track
      delegated awaitables; CPython matches outside an active suspend).
    - `gi_suspended` → True iff Started && !Done && !Running.
    - `cr_origin` → None (no origin tracking).
- `vm/ops.go` getAttr `*object.Generator` branch checks
  `genIntrospect` before falling through to `genMethod`.
- New fixture `internal/testdata/343_generator_introspection.py` —
  gi_* progression across just-created/suspended/done; coroutine
  cr_* incl. `cr_frame` going None on completion; ag_* on async
  generators; cross-prefix attrs report `hasattr(...) == False`;
  in-flight `gi_running == True` from inside the generator body.

Out of scope: PEP 667 FrameLocalsProxy (S1), `yield from` /
`await` chain tracking for the `*_yieldfrom` / `*_await` attrs,
`cr_origin` requires `sys.set_coroutine_origin_tracking_depth`.
