# v0.0.327

`pty` module gets a dedicated deep fixture (327) with missing re-exports
filled in.

The module was stubbed in fixture 320 with only the four integer constants
and three error-raising stubs. Missing were `close`, `waitpid`, `setraw`,
`tcgetattr`, and `tcsetattr` which CPython's `pty.py` re-exports from
`os` and `tty`.

New in this release:

- `close(fd)` -- no-op stub (re-export from os).
- `waitpid(pid, opts)` -- returns `(0, 0)` stub (re-export from os).
- `setraw(fd)` -- raises `OSError` on non-tty fd (re-export from tty).
- `tcgetattr(fd)` -- raises `OSError` on non-tty fd (re-export from tty).
- `tcsetattr(fd, when, attrs)` -- no-op stub (re-export from tty).

Fixture 327 covers all four integer constants, callable checks for all
eight function names, `openpty` happy-path with fd cleanup, `close` with
invalid fd, `setraw`/`tcgetattr` raise path, and `waitpid` return shape.
