# v0.0.332 — 2026-04-29

Threading correctness — no GIL, fine-grained locks

goipy maps `threading.Thread` and `multiprocessing.Process` onto real
Go goroutines, so two threads run Python bytecode in parallel on
multiple cores. This release closes the remaining concurrent-safety
gaps the audit in `notes/Spec/1400/1477_goipy_v0032_threading.md`
turned up. Locks are per-object (cheap when uncontended) and scalar
counters use `sync/atomic`; there is still no global GIL.

- `classEpoch` is now `atomic.Uint64`. Two threads mutating class
  dicts can no longer lose method-cache invalidations or read torn
  values.
- `object.Set` and `object.Frozenset` carry a `sync.RWMutex`; `Add`,
  `Discard`, `Contains`, `Len`, and the `Items` snapshot are all
  guarded. `Eq`/`Hash` for sets snapshot under the lock before
  iterating.
- `object.List` carries a `sync.Mutex` plus `Append`, `Extend`,
  `Snapshot`, `Len` helpers. The `LIST_APPEND` and `LIST_EXTEND`
  opcodes and the `list.append`/`extend`/`pop`/`insert`/`remove`/
  `clear`/`reverse` methods route through the lock. Direct `V` access
  remains valid for known-single-threaded contexts (frame
  construction, immediate iteration of just-built containers).
- `object.Bytearray.views` is `atomic.Int32` exposed via `AddView`,
  `DropView`, `HasViews`. Concurrent `memoryview()` create/release
  no longer races with resize-attempt checks.
- `object.Cell` carries a `sync.Mutex` and `Load`/`Store`/`Unset`
  helpers. `LOAD_DEREF`, `STORE_DEREF`, `DELETE_DEREF`, and
  `LOAD_FROM_DICT_OR_DEREF` route through it so closures captured by
  goroutine-backed threads (e.g. `nonlocal` reads inside a
  `Thread(target=fn)`) are safe.
- `weakref` module's internal `table` is now guarded by a closure-local
  `sync.Mutex`. `ref()`, `getweakrefcount()`, `getweakrefs()` lock
  around their map reads/writes.
- New fixture `internal/testdata/332_threading_stress.py` — 8 threads
  × 200 ops each, hammering shared list / dict / set / weakref
  without a Python-level Lock. Passes under `go test -race`.

Out of scope (still queued for later releases): `compile()`/`exec(str)`,
`__slots__`, true weakref via Go GC notification, pickle protocol-5
PickleBuffer, subprocess shell handling. Generators remain
single-thread by Python convention; cross-thread `gen.send()` is
undefined upstream and not addressed here.
