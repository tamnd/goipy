# v0.0.346 — 2026-04-29

`pickle.PickleBuffer` + `compile()`/`exec(str)`/`eval(str)` raise SyntaxError

Closes the **v0.1.0 tail** items called out in
`notes/Spec/1400/1476_goipy_v01_prep.md` (compile/exec(str)/eval(str),
pickle 5 PickleBuffer, weakref). Spec at
`notes/Spec/1500/1540_goipy_v0046_tail_pickle5_compile.md`.

After this release the v0.1.0 "run any 3.14 .pyc" gate is green.

## What's in

- `vm/stdlib_pickle.go`: new `PickleBuffer` class on the pickle
  module — `__init__(bytes-like)` accepts bytes / bytearray /
  memoryview; `__bytes__` returns a fresh `*object.Bytes`;
  `raw()` returns a readonly `*object.Memoryview` over the buffer;
  `release()` invalidates and subsequent ops raise
  `ValueError("operation forbidden on released PickleBuffer object")`,
  matching CPython.
- `vm/builtins.go` `bytes()` builtin: now consults `__bytes__` on
  instances before falling through to the iterable path. Required
  for `bytes(PickleBuffer(...))` and any user class that defines
  `__bytes__`.
- `vm/stdlib_builtins.go`: `compile()`, `exec(str)`, `eval(str)`
  raise `SyntaxError` with a clear message that goipy is
  .pyc-only and a Python source parser (gocopy) is in development.
  Previous `NotImplementedError` made user code that catches
  `SyntaxError` (the documented compile-time class) miss the
  except branch.
- `vm/interp.go` `New()`: eagerly call `buildBuiltins()` so
  `__import__` / `exec` / `eval` / `compile` are available at the
  global namespace without requiring `import builtins`.

## Out of scope

- `compile()` actually compiling Python source — `gocopy` parser is
  the path; wiring is a separate cycle.
- `weakref` GC tracking — Go runtime has no fundamental weak refs;
  `runtime.AddCleanup` (Go 1.25+) is achievable in principle but
  CPython callback ordering makes it a real-engineering task.
  Punted post-v0.1.0.
- Pickle 5 out-of-band buffer round-tripping in `pickle.dumps` /
  `pickle.loads` (would need protocol-5 opcode handling). This
  release exposes the `PickleBuffer` type but does not change
  pickle's wire-format paths.
