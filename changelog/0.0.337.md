# v0.0.337 — 2026-04-29

Frame & code exposure

The v0.0.334 deep gap audit
(`notes/Spec/1500/1530_goipy_v01_gap_audit.md`) flagged B3+F3:
`sys._getframe()` was missing, frame attributes weren't surfaced, and
`co_qualname`, `co_argcount`, `co_consts`, `co_varnames`, `co_lines()`
etc. weren't accessible. Without them, `inspect.currentframe()`
returned None, `inspect.stack()` returned `[]`, and the stdlib
`logging` module's `findCaller` (which uses `sys._getframe(2)` to skip
frames) collapsed `%(funcName)s` to `<unknown>`. Spec at
`notes/Spec/1500/1533_goipy_v0037_frame_code_exposure.md`.

- `sys._getframe([depth])` walks `i.curFrame.Back`; depth=0 is the
  caller's frame (builtins don't push their own frame).
- `*vm.Frame` is now Python-visible. A new `TypeNameHook` in
  `object/hooks.go` lets vm register the type name "frame" so
  `type(fr).__name__ == 'frame'` and the existing TypeName fallback
  still works for everything else.
- Frame attributes via `vm/ops.go`: `f_code`, `f_globals`,
  `f_builtins`, `f_locals`, `f_back`, `f_lineno`, `f_lasti`,
  `f_trace`, `f_trace_lines`, `f_trace_opcodes`. `f_locals` is the
  live dict for module/class-body frames; for function frames we
  materialise a snapshot from `Fast` slots and cells (PEP 667
  FrameLocalsProxy with write-back is deferred).
- Code attributes extended in `vm/ops.go`: `co_qualname`,
  `co_argcount`, `co_posonlyargcount`, `co_kwonlyargcount`,
  `co_nlocals`, `co_stacksize`, `co_flags`, `co_consts`,
  `co_names`, `co_varnames`, `co_freevars`, `co_cellvars`,
  `co_code`, `co_lnotab`, `co_linetable`, `co_lines()`. The
  `co_lines()` iterator yields a single `(start, end, line)` span
  using `FirstLineNo` — fine for most "give me *some* attribution"
  consumers; precise per-op resolution awaits a line-table decoder.
- `types.FrameType` and `types.TracebackType` registered in
  `vm/stdlib_types.go` so `isinstance(fr, types.FrameType)` works
  and `inspect.isframe` (previously a hardcoded `false`) now does
  the right check.
- `inspect.stack`, `getouterframes`, `getframeinfo`, `getlineno`, and
  `currentframe` (`vm/stdlib_inspect.go`) walk the real frame chain
  instead of returning empty stubs.
- New fixture `internal/testdata/337_frame_code_exposure.py` —
  exercises `_getframe`, `f_back` chain, `inspect.stack`,
  `co_qualname`/`co_consts`/etc., and `co_lines()`.

Out of scope: FrameLocalsProxy write-back (PEP 667), full PEP 626
line-table decoding (needs its own release before `co_positions()`,
PEP 657, can land), `f_trace` writeback (lands with `sys.settrace` in
v0.0.345).
