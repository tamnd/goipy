# v0.0.328

`fcntl` module gets a dedicated deep fixture (328) with constant and
function return-type fixes.

The module was stubbed in fixture 320 with 31 constants and 4 stubs that
all returned `None`. Several constants were wrong, missing, or not present
in CPython.

Constant fixes:

- `F_RDAHEAD`: 60 -> 45 (wrong value)
- Added `F_GETLEASE=107`, `F_SETLEASE=106`, `F_GETPATH=50`,
  `F_OFD_GETLK=92`, `F_OFD_SETLK=90`, `F_OFD_SETLKW=91`
- Removed `F_FREEZE_FS`, `F_PREALLOCATE`, `F_RDADVISE`, `F_SETSIZE`,
  `F_THAW_FS` (not present in CPython's fcntl module)

Function fixes:

- `fcntl(fd, cmd[, arg])` now returns `int` (was `None`).
- `ioctl(fd, request[, arg])` now returns `int` when called without a
  buffer arg, or `bytes` when called with a `bytes` arg (was `None`).

Fixture 328 asserts all 31 macOS constants with exact values, verifies
the five removed constants are absent, callable checks for all 4
functions, `fcntl()` int return type, `ioctl()` int/bytes return type,
and flock/lockf raise-or-None paths.
