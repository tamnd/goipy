# v0.0.341 — 2026-04-29

Encoding error handlers + `bytes.hex(sep, group)`

The v0.1.0 gap audit (`notes/Spec/1500/1530_goipy_v01_gap_audit.md`)
flagged C1 (`str.encode` ignored its `errors=` kwarg) and C2
(`bytes.hex` ignored its separator + group args). Both produced
silently wrong results -- `'中'.encode('ascii', 'replace')` returned
the underlying utf-8 bytes instead of `b'?'`, and
`b'\xaa\xbb'.hex('-')` returned `'aabb'` instead of `'aa-bb'`. Spec
at `notes/Spec/1500/1535_goipy_v0041_encoding_handlers_bytes_hex.md`.

- `vm/call.go` `strMethod.encode` now parses the
  `(encoding='utf-8', errors='strict')` signature and dispatches
  through `i.codecsEncode`, which already implements every error
  handler the test corpus exercises.
- `vm/call.go` `bytesMethod.decode` and `bytearrayMethod.decode`
  parse the same signature and dispatch through `i.codecsDecode`.
  Shared parsing in the new `bytesDecodeArgs` helper.
- `vm/call.go` `bytesSubMethod.hex` (shared by `Bytes` and
  `Bytearray`) accepts `sep` and `bytes_per_sep` positional args
  via the new `bytesHexFmt`. Positive group counts from the right;
  negative groups from the left; missing/empty sep keeps the
  original fast path.
- `vm/stdlib_codecs.go`: encoding errors now raise the proper
  `UnicodeEncodeError` / `UnicodeDecodeError` subclasses (used to
  raise the parent `UnicodeError`, so narrow `except
  UnicodeEncodeError` clauses missed).
- `vm/stdlib_codecs.go` `applyEncodeErrors` adds the `namereplace`
  handler — uses `unicodedata.name` when available, falls back to
  the `\uXXXX` / `\UXXXXXXXX` form.
- New fixture `internal/testdata/341_encoding_hex.py` —
  ascii/replace/ignore/xmlcharrefreplace/backslashreplace/
  namereplace, latin-1, default utf-8, `encoding=` kwarg, decode
  round-trips, hex with sep, hex with positive/negative group,
  bytearray.hex, empty-bytes hex.

Out of scope: `surrogateescape` / `surrogatepass` round-trip
(would require allowing isolated surrogates in goipy's utf-8 string
representation), `codecs.register_error` for user-registered
handlers — punted to a separate ticket.
