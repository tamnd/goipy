# v0.0.338 — 2026-04-29

PEP 585 / 604 / 560 — subscriptable builtins, union types, `__mro_entries__`

The v0.1.0 gap audit (`notes/Spec/1500/1530_goipy_v01_gap_audit.md`)
flagged three blockers/correctness gaps that all hit the same
generic-alias plumbing:

- **B5** — `list[int]`, `dict[str, int]`, … raised
  `TypeError 'function' object is not subscriptable` because
  `list`/`dict`/etc. are `*object.BuiltinFunc` values without
  `__class_getitem__`.
- **B6** — `int | str` raised
  `TypeError unsupported operand type(s) for |` because `bitop("|")`
  ultimately fell through to a bigint path.
- **C4** — `class C(g)` where `g.__mro_entries__` returns `(object,)`
  silently dropped the substituted bases; `__build_class__` filtered
  for `*object.Class` and never invoked `__mro_entries__`.

Spec at `notes/Spec/1500/1532_goipy_v0038_pep585_604_560.md`.

- New `vm/genericalias.go` defines two singleton classes:
  - `types.GenericAlias` instances carry `__origin__` + `__args__`,
    expose `__repr__` (`list[int]`), `__getitem__` for
    re-subscription, `__or__`/`__ror__`, `__mro_entries__` (returns
    `(__origin__,)` so `class C(list[int])` inherits from `list`),
    and `__call__` (forwards to origin so `list[int]([1,2])` works).
  - `types.UnionType` instances carry a flattened, deduped
    `__args__` tuple. `__repr__` joins with ` | `, `__or__`/`__ror__`
    extend the union, and `__instancecheck__` walks members.
- `installTypeSubscriptHooks` populates each builtin's `Attrs`
  with `__class_getitem__`, `__or__`, `__ror__`. Wired on `list`,
  `dict`, `tuple`, `set`, `frozenset`, `type`, `int`, `float`,
  `complex`, `str`, `bytes`, `bytearray`, `bool`. The `object` class
  picks up `__or__`/`__ror__` via `installClassUnionHooks`.
- `vm/binops.go` `getitem` checks `BuiltinFunc.Attrs` for
  `__class_getitem__` so `list[int]` reaches the new path.
- `vm/binops.go` `bitop` adds an `isTypeLike` short-circuit before
  the bigint fallback: when both operands are class-like
  (`*object.Class`, type-decorated `*object.BuiltinFunc`, NoneType,
  GenericAlias, UnionType), `|` builds a UnionType.
- `vm/builtins.go` `__build_class__` walks the raw bases and calls
  `__mro_entries__(bases)` on any non-class entry, splicing the
  returned tuple in place. `lookupMROEntries` finds the method on
  Instance / Class / BuiltinFunc.
- `vm/builtins.go` `isinstance` short-circuits for UnionType:
  walks `__args__` recursively. (Full metaclass `__instancecheck__`
  dispatch is gap C3, deferred to v0.0.342.)
- New fixture `internal/testdata/338_pep585_604_560.py` — 9 sections
  covering subscriptable list/dict, re-subscription, `int | str`
  membership, three-way union flattening, `list[int] | None`,
  `__mro_entries__` substitution, user-class unions, and a
  `typing.List[int]` regression check.

Out of scope: subclassing `list[int]` actually inheriting list
semantics (would need first-class container classes — out of v0.1.0
scope), exposing `types.GenericAlias`/`types.UnionType` on the
`types` module (no real consumers ask for them yet).
