# v0.0.326

`tty` module gets a dedicated deep fixture (326) with several missing
pieces filled in.

New in this release:

- Struct-index constants added: `IFLAG=0`, `OFLAG=1`, `CFLAG=2`,
  `LFLAG=3`, `ISPEED=4`, `OSPEED=5`, `CC=6`. These are defined in
  `tty.py` directly (not re-exported from termios).
- `error` added as an alias for `OSError` (re-exported from termios).
- `cfmakeraw(mode)` implemented: clears all POSIX input-mode flags,
  disables `OPOST`, clears `PARENB`/`CSIZE` and sets `CS8`, clears
  all local-mode flags, and sets `CC[VMIN]=1` / `CC[VTIME]=0`.
- `cfmakecbreak(mode)` implemented: clears `ECHO` and `ICANON`,
  sets `CC[VMIN]=1` / `CC[VTIME]=0`.
- `tcgetwinsize` and `tcsetwinsize` stubs added (raise `OSError` on
  non-tty, consistent with the other tty ioctl stubs).
- `setraw` and `setcbreak` now properly raise `OSError` (they
  previously returned `None`, hiding the fact that `tcgetattr` would
  fail on a non-tty fd).
- `tcgetattr` now raises `OSError` instead of returning a fake list,
  matching real behaviour on non-tty fds.

Fixture 326 verifies all struct-index constants, a key sample of
re-exported termios constants, all 12 callable names, in-place mutation
by `cfmakeraw` and `cfmakecbreak` (bit clearing and `CC` update), and
the `OSError` raise path for `setraw`, `setcbreak`, `tcgetattr`, and
`tcgetwinsize`.
