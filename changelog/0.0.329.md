# v0.0.329

`resource` module gets a dedicated deep fixture (329) and one type fix.

The module was stubbed in fixture 320 with all constants and functions
present, but `struct_rusage` initialised `ru_utime` and `ru_stime` as
`int(0)` instead of `float(0.0)`. CPython always returns floats for
these two time fields.

Fix:

- `struct_rusage.ru_utime` and `ru_stime` now default to `float(0.0)`
  in both `mkRusage()` and `__init__`. All other ru_* fields remain
  `int`.

Fixture 329 covers all 13 constants with exact values, `error` exception
check, `struct_rusage` field types (`float` for `ru_utime`/`ru_stime`,
`int` for the 14 counter fields), `getrusage` for both `RUSAGE_SELF` and
`RUSAGE_CHILDREN`, `getrlimit` tuple shape for three resource types,
`setrlimit` callable, and `getpagesize() > 0`.
