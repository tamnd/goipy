# v0.0.331 — 2026-04-28

v0.1.0 prep: dispatch + intrinsics + MRO

First iteration toward "run any .pyc produced by Python 3.14". A deep
gap analysis in `notes/Spec/1400/1476_goipy_v01_prep.md` enumerates the
remaining v0.1.0 blockers; this release ships the highest-leverage
subset.

- C3 MRO linearization in place of depth-first traversal. Diamond
  inheritance now resolves methods in CPython-compatible order. Falls
  back to depth-first if no consistent linearization exists, so
  ambiguous fixtures don't crash.
- All six previously stubbed `INTRINSIC_1` cases:
  `IMPORT_STAR` (with `__all__` honoring), `TYPEVAR`, `PARAMSPEC`,
  `TYPEVARTUPLE`, `SUBSCRIPT_GENERIC`, `TYPEALIAS`.
- All five `INTRINSIC_2` cases: `PREP_RERAISE_STAR` (PEP 654 `except*`),
  `TYPEVAR_WITH_BOUND`, `TYPEVAR_WITH_CONSTRAINTS`,
  `SET_FUNCTION_TYPE_PARAMS`, `SET_TYPEPARAM_DEFAULT`. Constants added
  to `op/opcodes.go`.
- 20 specialized opcodes routed to their unspecialized counterparts so
  PEP 659 inline-cache-specialized .pyc files don't crash:
  13 `LOAD_ATTR_*`, 3 `STORE_ATTR_*`, `SEND_GEN`, `FOR_ITER_GEN`,
  `BINARY_OP_EXTEND`, `CALL_ALLOC_AND_ENTER_INIT`.
- `CHECK_EG_MATCH` now properly splits ExceptionGroups by handler type
  instead of treating them like plain `CHECK_EXC_MATCH`.
- `__build_class__` copies closure cells into freevar slots so PEP 695
  generic class bodies can resolve `.type_params` via
  `LOAD_FROM_DICT_OR_DEREF`.
- `Class.__mro__` attribute exposed.

Out of scope for this release (tracked in 1476): `compile()`/`exec(str)`,
`__slots__`, true weakref, threading correctness audit, pickle protocol
5 PickleBuffer, subprocess shell handling.
