# v0.0.318 - 2026-04-28

Dedicated fixture for winreg, fixture 318. Full coverage of
https://docs.python.org/3/library/winreg.html.

Fixture 316 added a basic winreg stub. This release fills the gaps and
adds a thorough standalone test.

Changes to vm/stdlib_winreg.go:

- Added 17 undocumented REG_* constants that are present on real Windows
  and listed in typeshed: REG_CREATED_NEW_KEY, REG_LEGAL_CHANGE_FILTER,
  REG_LEGAL_OPTION, REG_NOTIFY_CHANGE_*, REG_NO_LAZY_FLUSH,
  REG_OPENED_EXISTING_KEY, REG_OPTION_*, REG_REFRESH_HIVE,
  REG_WHOLE_HIVE_VOLATILE
- Added __enter__ (returns self) and __exit__ (returns None) to HKEYType
  so it works as a context manager
- Added __hash__ to HKEYType returning 0

New internal/testdata/318_winreg_deep.py tests:

- All 7 HKEY_* root constants by value and int type
- All 12 KEY_* access constants by value
- All 14 documented REG_* value type constants
- All 17 undocumented REG_* constants
- error is OSError
- HKEYType is a type; instance creation, handle attribute, Close/Detach
- HKEYType __bool__ and __int__
- HKEYType as context manager via with statement
- All 23 registry functions callable
- OpenKey, QueryValue, CreateKey all raise OSError when invoked

Expected output crafted from known implementation values since CPython
cannot import winreg on non-Windows.
