# v0.0.323

`pwd` module gets a dedicated deep fixture (323) and two fixes.

The module was stubbed in fixture 320 but `getpwnam` always raised
`KeyError` and `getpwall` returned an empty list, making common patterns
like `getpwnam('root')` and iterating `getpwall()` fail.

Fixes:

- `getpwnam('root')` now returns the root stub entry instead of raising
  `KeyError`.
- `getpwall()` now returns a list containing the root entry so
  `len(getpwall()) >= 1`.

Fixture 323 covers `struct_passwd` field types, `getpwuid`/`getpwnam`
happy paths, `KeyError` on unknown uid/name, `getpwall`, and the
uid round-trip `getpwuid(getpwnam('root').pw_uid).pw_name == 'root'`.
