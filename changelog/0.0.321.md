# v0.0.321

`shlex` module gets a full-coverage deep fixture (321) and two fixes.

The module already had `split`, `join`, and `quote` wired up -- what was missing
was the `shlex` class itself and a bug in the lexer for non-posix mode.

New in this release:

- `shlex.shlex` class with correct default attribute values: `wordchars`,
  `whitespace`, `escape`, `quotes`, `commenters`, `whitespace_split`, `debug`,
  `lineno`, `token`, `punctuation_chars`, `posix`, `eof`, `source`, `state`,
  `filestack`, `pushback`, `infile`.  Constructor accepts `posix=` and
  `punctuation_chars=` keyword args.
- `shlex` instance methods: `get_token`, `read_token`, `push_token`,
  `push_source`, `pop_source`, `sourcehook`, `error_leader`, `__iter__`,
  `__next__`.

Bug fixes:

- `shlex.split(..., posix=False)` -- the lexer now preserves quote characters
  when posix mode is off (previously quotes were always stripped).
- Kwarg handling in `split()` switched to `isTruthy` so non-Bool falsy values
  (e.g. int 0) are also handled correctly.

Fixture 321 covers split, join, quote and the shlex class with 86 assertions.
