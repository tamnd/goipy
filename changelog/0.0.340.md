# v0.0.340 — 2026-04-29

Exception API completion (PEP 654 / PEP 678)

The v0.1.0 gap audit (`notes/Spec/1500/1530_goipy_v01_gap_audit.md`)
flagged F1: `*object.Exception` was missing `add_note`,
`with_traceback`, `__suppress_context__`, and the PEP 654
`BaseExceptionGroup.split` / `subgroup` / `derive` API. Modern
asyncio code that filters a `TaskGroup`-raised group via
`g.split(ValueError)` blew up on `AttributeError`. Spec at
`notes/Spec/1500/1534_goipy_v0040_exception_api.md`.

- New `vm/exc_methods.go` `installExceptionMethods` puts
  `add_note(s)` and `with_traceback(tb)` on `BaseException.Dict`
  (so every subclass inherits via the existing `classLookup` →
  `BoundMethod` path) and `split` / `subgroup` / `derive` on
  `BaseExceptionGroup.Dict`.
- `add_note` lazily allocates `e.Dict["__notes__"]` as a list and
  appends; `with_traceback` accepts None or a `*object.Traceback`,
  assigns to `e.Traceback`, returns self.
- `vm/ops.go` Exception `getAttr` returns
  `__suppress_context__` as `False` when unset (reads `e.Dict`
  otherwise) and exposes group-only `message` / `exceptions`
  attributes when the class is a `BaseExceptionGroup` subclass.
- `vm/ops.go` `setAttr` now handles `*object.Exception`. Plain
  attributes flow into `e.Dict`; `__traceback__` / `__cause__` /
  `__context__` are routed to the typed fields. Setting
  `__cause__` implicitly sets `__suppress_context__` to True
  (matches CPython's `raise ... from`).
- `vm/call.go` constructor: `BaseExceptionGroup(msg, excs)` auto-
  promotes to `ExceptionGroup` when every inner exception is an
  `Exception` subclass, matching PEP 654. Inner-exception sequence
  is normalized to a tuple so `eg.exceptions` is a tuple.
- `vm/exc_methods.go` `egSplit` recursively partitions a group:
  inner groups are split in-place, the matching half is rebuilt
  via `derive`, top-level cause/context/traceback/__notes__ are
  copied onto both sub-groups. `subgroup` reuses `egSplit` and
  returns just the matched half. `egMatches` accepts a class, a
  tuple of classes, or a callable predicate.
- New fixture `internal/testdata/340_exception_api.py` —
  add_note + __notes__, with_traceback, __suppress_context__,
  group split (class & callable), subgroup, derive,
  except*-handler note round-trip.

Out of scope: recursive cause/context/traceback copy onto every
nested sub-group (gap audit lists this as nice-to-have; current
top-level copy matches the common patterns), strict
`BaseExceptionGroup.__init__` validation that rejects non-
exception inners.
