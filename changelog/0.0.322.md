# v0.0.322

`posix` module gets deep coverage with fixture 322 -- filling in everything that
was stubbed or missing after the bulk Unix fixture (320).

New in this release:

- Missing constants added: `POSIX_SPAWN_OPEN`, `POSIX_SPAWN_CLOSE`,
  `POSIX_SPAWN_DUP2`; `PRIO_DARWIN_PROCESS`, `PRIO_DARWIN_THREAD`,
  `PRIO_DARWIN_BG`, `PRIO_DARWIN_NONUI`; `P_ALL`, `P_PID`, `P_PGID`;
  `ST_RDONLY`, `ST_NOSUID`; `TMP_MAX`; `WCONTINUED`, `WEXITED`, `WNOWAIT`,
  `WSTOPPED`; `SEEK_HOLE`, `SEEK_DATA`; `O_EXEC`, `O_SEARCH`,
  `O_NOFOLLOW_ANY`.
- Named result classes: `stat_result`, `statvfs_result`, `uname_result`,
  `times_result`, `terminal_size`, `waitid_result`, `DirEntry` -- each
  exposes the expected named attributes on instances.
- `stat()`, `lstat()`, `fstat()` now return a `stat_result` instance with
  zeroed fields instead of `None`, so code like `os.stat('/').st_size` no
  longer raises `AttributeError`.
- `statvfs()` / `fstatvfs()` return a `statvfs_result` instance.
- `uname()` returns a `uname_result` with Darwin defaults.
- `times()` returns a `times_result` with 0.0 fields.
- `get_terminal_size()` returns a `terminal_size` instance.
- New functions: `fspath`, `strerror`, `device_encoding`, `umask`, `system`,
  `closerange`, `get_blocking`, `set_blocking`, `get_inheritable`,
  `set_inheritable`, `major`, `minor`, `makedev`, `WSTOPSIG`,
  `waitstatus_to_exitcode`, `sched_get_priority_max`,
  `sched_get_priority_min`, `sched_yield`, `sync`, `setpgrp`,
  `login_tty` (raises `OSError`), `grantpt`, `posix_openpt`, `ptsname`,
  `unlockpt`, `register_at_fork`, `getgrouplist`, `initgroups`, `chroot`,
  `posix_spawn`, `posix_spawnp`, `preadv`, `pwritev`, `readinto`.

Fixture 322 covers all new constants, all named result classes, and all
callable functions with 100+ assertions.
