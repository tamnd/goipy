# v0.0.324

`grp` module gets a dedicated deep fixture (324) and two fixes.

The module was stubbed in fixture 320 but `getgrgid` and `getgrnam` always
raised `KeyError` and `getgrall` returned an empty list, making any real
usage fail.

Fixes:

- `getgrgid(0)` now returns the wheel stub entry
  (`gr_name='wheel'`, `gr_gid=0`, `gr_mem=['root']`).
- `getgrnam('wheel')` now returns the same wheel stub.
- `getgrall()` now returns `[wheel_entry]` so `len(getgrall()) >= 1`.

Fixture 324 covers `struct_group` field types, `getgrgid`/`getgrnam`
happy paths, `KeyError` on unknown gid/name, `getgrall`, and both the
gid and name round-trips.
