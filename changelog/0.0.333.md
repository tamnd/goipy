# v0.0.333 — 2026-04-28

`__slots__` semantics

The v0.0.332 changelog flagged `__slots__` as still queued. This
release closes that gap. goipy doesn't chase CPython's memory-layout
optimisation — slot values still live in the instance's underlying
dict — but the user-visible contract is now enforced. Spec at
`notes/Spec/1400/1478_goipy_v0033_slots.md`.

- `__build_class__` reads `__slots__` from the class body namespace
  during construction. Tuple, list, single-string, and dict forms all
  parse to the same canonical name list. Dict form keeps keys (the
  values are docstrings, ignored).
- `Class` gained two fields: `Slots []string` (own declared names,
  excluding `'__dict__'` and `'__weakref__'`) and `NoDict bool` (true
  when the class declares `__slots__` and neither the slot list nor
  any base class re-enables `__dict__`).
- `setAttr` and `delAttr` reject non-slot names on `NoDict` instances
  with `AttributeError`, after the descriptor and `__setattr__` /
  `__delete__` overrides have had their say. The check walks the MRO
  so subclasses inherit their parents' slots.
- `getAttr` returns `AttributeError` for `__dict__` on `NoDict`
  instances. When any class in the MRO declared `__slots__` but the
  instance still has a dict (e.g. a subclass without slots, or
  `'__dict__'` listed in slots), `__dict__` returns a filtered
  snapshot that excludes slot names — so `obj.__dict__` shows only
  the dynamic non-slot attributes, matching CPython.
- `'__dict__'` listed inside `__slots__` re-enables the dict path on
  the class. `'__weakref__'` is accepted as a slot name and silently
  inert (goipy's weakrefs already work on every instance).
- New fixture `internal/testdata/333_slots.py` covers all four slot
  syntaxes, `del` on slot/non-slot, subclass-extends-slots,
  subclass-without-slots-regains-dict, `'__dict__'` opt-in,
  `__slots__ = ()`, and `'__weakref__'` opt-in. Verified against
  CPython 3.14.

Out of scope: per-slot member descriptors (and the matching memory
savings), `__slots__` collisions with class-body defaults raising at
class creation, and the read-only-cell semantics CPython's slot
descriptors give. Goipy's slots are a contract over a regular dict,
not a separate storage area.
