# v0.0.345 — 2026-04-29

`sys.settrace` / `sys.setprofile` (F7)

Closes group **F7** from the v0.1.0 audit
(`notes/Spec/1500/1530_goipy_v01_gap_audit.md`). Spec at
`notes/Spec/1500/1539_goipy_v0045_settrace_setprofile.md`.

`pdb`, `coverage.py`, and third-party profilers (`yappi`,
`line_profiler`) drive their hooks through `sys.settrace` /
`sys.setprofile`. Goipy had stub no-ops in `threading` and `pdb` and
nothing on `sys` itself; tracers silently never fired.

- `vm/interp.go`: `Interp.traceFn` / `Interp.profileFn` /
  `Interp.inTrace` (re-entry guard so the tracer's own dispatch
  doesn't recurse). New `runFrame` hooks fire `'call'` /
  `'return'` / `'exception'` events; the global trace's `'call'`
  return value becomes the per-frame `LocalTrace`.
- `vm/frame.go`: `Frame.LocalTrace` and `Frame.LastLine` track
  per-frame trace state. `LastLine` is seeded to `Code.FirstLineNo`
  on frame entry so the RESUME-on-def-line opcode doesn't fire a
  spurious line event before the first body line — matching
  CPython 3.11+.
- `vm/dispatch.go`: hot-loop guard at top of dispatch checks
  `f.LocalTrace != nil` (nil for every frame when tracing is off
  → near-zero overhead) and fires `'line'` on source-line
  transitions via `i.fireLineEvent`. Per CPython, a local trace
  returning None disables further line events on that frame; a
  different callable replaces the local trace.
- `vm/call.go` frame pool reuse path resets `LocalTrace` /
  `LastLine` so a recycled frame doesn't carry old state.
- `vm/ops.go`: `Frame` `f_trace` getter returns `LocalTrace`;
  setter writes to `LocalTrace`. `f_trace_lines` / `f_trace_opcodes`
  setters accept-and-ignore.
- `vm/stdlib_sys.go`: `sys.settrace`, `sys.gettrace`,
  `sys.setprofile`, `sys.getprofile`. None disables; non-None
  must be callable (caller error surfaces at first invocation).
- New fixture `internal/testdata/345_settrace_setprofile.py` —
  settrace/gettrace round-trip, `('f','call')`/`('f','line')*N`/
  `('f','return')` event sequence, per-frame f_trace replacement
  (global tracer returns a different fn for `g`'s call), profile
  events without lines, disabling tracer mid-execution.

Out of scope: `'opcode'` event (rarely used; `f_trace_opcodes`
stays a no-op); `sys.monitoring` (PEP 669) — separate audit
entry; `frame.f_lineno = N` jump during pdb.
