# v0.0.335 — 2026-04-29

Function introspection

The v0.0.334 deep gap audit
(`notes/Spec/1500/1530_goipy_v01_gap_audit.md`) flagged B1+B2:
`function.__code__`, `__defaults__`, `__kwdefaults__`,
`__annotations__`, `__qualname__`, `__closure__`, `__globals__`,
`__module__`, `__dict__` were never surfaced through `getAttr` even
though every field already lived on the `*object.Function` struct.
Without them, `functools.wraps`, `inspect.signature`, `dataclasses`,
`typing.get_type_hints`, `pytest.parametrize`, `click`, `fastapi` and
`pydantic` all crashed on the very first introspection call. Spec at
`notes/Spec/1500/1531_goipy_v0035_function_introspection.md`.

- `vm/ops.go:getAttr` extended Function branch covers all eleven
  attributes. `__module__` is derived from `globals()['__name__']` —
  CPython does the same when no explicit module is recorded.
  `__annotations__` and `__dict__` lazy-init when first read.
- PEP 649 `__annotate__` is now wired. `SET_FUNCTION_ATTRIBUTE` flag
  `0x10` (Python 3.14's lazy annotation function) lands on a new
  `Function.Annotate` field. The first read of `__annotations__` calls
  `__annotate__(1)` (VALUE format) and caches the result. Direct
  `__annotate__` access also works.
- Bound methods now forward attribute lookups to their underlying
  function (`vm/ops.go` BoundMethod branch). `m.__defaults__`,
  `m.__name__`, `m.__doc__` work; `m.__self__` and `m.__func__` are
  also exposed.
- New fixture `internal/testdata/335_function_introspection.py` —
  walks every attribute, confirms None for absent defaults/closure,
  exercises closure cells, dict round-trip, BoundMethod forwarding,
  and `typing.get_type_hints` integration.

Out of scope: setter for `__code__` / `__defaults__` / `__qualname__`
(CPython lets you write to these — punt); `__type_params__`
(PEP 695, generic functions) — covered when generic functions land.
